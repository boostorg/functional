Identity

The header functional.hpp provides a function object type whose operator() returns its argument unchanged:

  • identity

This is a C++03 implementation of C++20's std::identity.

Usage

Identity serves as the default projection in constrained algorithms. Its direct usage is usually not needed. For example,

template<typename Projection = boost::identity>
struct projected_range_printer
{
    template<typename Iterator>
    void print(Iterator b, Iterator e, Projection proj = Projection()) { ... }
};
...
std::vector<std::string> c;
...
projected_range_printer<>().print(c.begin(), c.end());

Argument And Return Types

The "ISO/IEC 14882:2020" C++ Standard defines identity like this:

struct identity {
  template<class T>
  constexpr T&& operator()(T&& t) const noexcept;
  using is_transparent = unspecified ;
};

Additional Types Defined

is_transparent

This member type indicates to the caller that this function object is a transparent function object: it accepts arguments of arbitrary types and uses perfect forwarding(or its emulation in C++03), which avoids unnecessary copying and conversion when the function object is used in heterogeneous context, or with rvalue arguments. In particular, template functions such as std::set::find and std::set::lower_bound make use of this member type on their Compare types.
